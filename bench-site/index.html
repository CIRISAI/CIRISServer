In plain language

  • One CPU core ingests ~4,946 fully post-quantum-signed events every second — signature verification included.
  • A single core fans out 2,000 live streams at 30 fps using 1.6% of its capacity — scale comes from adding people, not datacenters.
  • Data shared on one node appears on another across the real mesh in ~6.5 ms, and out-of-group nodes receive nothing.
  • Scoring a participant's real-time coherence takes 49 microseconds.

The honest cost of post-quantum proof measured

We don't hide this: a post-quantum signature is real work. Per signed event, the hybrid Ed25519 + ML-DSA-65 sign+verify is 550 µs vs 35.9 µs for classical Ed25519 alone — about 15×. ML-DSA dominates; that's the price of being quantum-resistant today, measured rather than waved away. Even so, a single core still ingests ~4,946 of these signed events per second — the verify is one stage of a pipeline, and throughput stays high.

How it holds up

Peers relay for peers

A small relay tree forwards traffic between participants, so capacity grows as people join instead of requiring bigger servers. The relays forward sealed bytes they can never read.

Post-quantum end-to-end

Every event and stream is signed and encrypted with hybrid classical + post-quantum cryptography (Ed25519+ML-DSA, X25519+ML-KEM). The provenance survives a future quantum computer.

Redundant, recoverable storage

The corpus is split into fountain-coded shares spread across holders, so it rebuilds even as a large fraction of them go offline — the resilience table above is measured, not assumed.
